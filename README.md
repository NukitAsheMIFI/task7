Task 7: Копирование файла в два параллельных процесса с помощью fork

Описание

Данный проект демонстрирует работу с параллельными процессами на языке C с использованием системного вызова fork.

Программа открывает общий файл на чтение, создаёт параллельный процесс с помощью fork, после чего оба процесса (родительский и дочерний) читают информацию из общего файла и копируют её в собственные выходные файлы. По окончании копирования программа выводит на экран содержимое полученных файлов.

Параметры командной строки

Путь до файла (один обязательный аргумент).

Входные данные

Ввод в стандартный поток отсутствует.

Выходные данные

Программа создаёт два файла:
- parent_copy (копия исходного файла, созданная родительским процессом)
- child_copy (копия исходного файла, созданная дочерним процессом)

После копирования программа выводит содержимое файлов в стандартный поток:
- Parent: <содержимое>
- Child: <содержимое>

Также программа выводит отладочную информацию о PID процессов.

Пример запуска

# ./task7 test_file

>Parent: My PID -- 1234
>Parent: My child's PID -- 1235
>Parent: Waiting for my child to call exit()...
>Child: My PID -- 1235
>Child: My parent's PID -- 1234
>Child: do job
>Child: Exit
>Parent: Child's exit status: 0
>Parent: do job
>Parent: Exit

# ls

>task7 test_file parent_copy child_copy

# cat parent_copy

>qwertyuiop...

# cat child_copy

>qwertyuiop...

Функциональные требования

Открытие общего файла: программа открывает файл, путь к которому передан в аргументе командной строки, в режиме чтения.

Создание параллельного процесса: с помощью системного вызова fork создаётся дочерний процесс.

Копирование в родительском процессе: родительский процесс ожидает завершения дочернего процесса через wait, затем читает данные из общего файла и записывает их в файл parent_copy. После этого выводит содержимое parent_copy на экран с префиксом "Parent: ".

Копирование в дочернем процессе: дочерний процесс читает данные из общего файла и записывает их в файл child_copy. После этого выводит содержимое child_copy на экран с префиксом "Child: ".

Сброс указателя файла: перед чтением общего файла каждый процесс вызывает rewind, чтобы начать чтение с начала файла.

Обработка ошибок: при невозможности открыть файлы программа выводит сообщение об ошибке через perror и завершается с соответствующим кодом.

Система сборки: GNU Make.

Технические подробности реализации

Системный вызов fork: создаёт новый процесс путём копирования вызывающего процесса. Родительскому процессу возвращается PID дочернего процесса, дочернему — 0, при ошибке — -1.

Функция wait: приостанавливает выполнение родительского процесса до тех пор, пока дочерний процесс не завершится. Позволяет получить статус завершения дочернего процесса.

Макросы для анализа статуса завершения:
- WIFEXITED(status): возвращает истину, если процесс завершился нормально
- WEXITSTATUS(status): возвращает код завершения процесса

Функция rewind: устанавливает указатель позиции в файле в начало. Используется для того, чтобы каждый процесс начинал чтение с первого байта файла.

Функция fread: читает данные из файла в буфер. Возвращает количество прочитанных байт.

Функция fwrite: записывает данные из буфера в файл.

Функция fopen: открывает файл с указанным режимом:
- "r" — чтение
- "w+" — чтение и запись (создаёт файл, если его нет, или обрезает до нуля)

Обработка ошибок: при ошибках открытия файлов, создания процесса или операций ввода-вывода программа выводит сообщение через perror и возвращает код ошибки errno.

Особенности работы с общим файлом: поскольку оба процесса читают из одного файла, важно, чтобы каждый процесс начинал чтение с начала. Для этого перед каждым чтением вызывается rewind. Без этого после завершения одного процесса указатель файла мог бы остаться в конце, и второй процесс не прочитал бы данные.
